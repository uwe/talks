\subsection{How do I spot a segfault?}

\frame{
  \frametitle{How do I spot a segfault?}

  \begin{itemize}
    \item look at the last few lines
    \item if it stops immediately, it might be a segfault
\pause
    \item grep for your script name
    \item output can be very large, with long lines
    \item {\tt grep -v site\_perl}
\pause
    \item in my case: buggy MSSQL driver (easysoft)
  \end{itemize}
}
